CHANGELOG
----------

Project: cmd-extended
Date: 7-19-2026
Contributors: Roberto J Dohnert

This file lists the recent updates by me to the following applications.

accountinfo
- Fixed user listing so it now reads every page of local accounts, not just the first page.
- Improved output flow so headers and "no accounts found" behavior are clearer.

app-events
- Added command options to filter logs:
  --count, --level, --provider, --since-hours, --no-message, --save, --help
- Added faster event querying and safer argument handling.
- Added provider metadata caching to reduce repeated expensive lookups.
- Fixed wide-string error printing and improved save-to-file flow.

calendar
- Replaced shell-based screen clear with a native Windows console clear.
- Added stronger numeric input validation to prevent broken input state.
- Improved invalid input recovery prompts.

cpuinfo
- Added --json output mode for automation.
- Replaced fixed-size registry reads with dynamic buffer reads to avoid truncation.
- Refactored logical thread bit counting into a helper for cleaner topology logic.

driveinfo
- Added --json output mode.
- Fixed drive-type logic by separating base drive type (enum) from display text.
- Corrected no-media detection for removable and optical drives.

file-manager
- Migrated core path and file operations from ANSI APIs to Unicode (W) APIs.
- Added path conversion helpers and wide-path normalization.
- Updated file and directory operations (cd, mkdir, rmdir, rm, copy, move) to use wide APIs.
- Improved display and command parsing helpers for more reliable behavior.

list-bluetooth
- Added UTF-8 console output setup and locale initialization for cleaner text output.

list-pci
- Added UTF-8 console output setup and locale initialization.
- Added explicit enumeration termination error check so real errors are reported.

list-usb
- Added UTF-8 console output setup and locale initialization.
- Fixed USB interface detail retrieval using the proper two-step SetupDi pattern.
- Added better per-device error reporting and loop termination validation.

meminfo
- Added --json output mode with physical and virtual memory fields.

reboot
- Hardened uppercase conversion to avoid unsafe char handling.
- Added --help support and unknown-option handling.
- Removed shell fallback reboot call; now uses API path with explicit error returns.

security-events
- Added command options to filter logs:
  --count, --level, --provider, --since-hours, --no-message, --save, --help
- Added faster event querying and safer argument handling.
- Added provider metadata caching to reduce repeated expensive lookups.
- Fixed wide-string error printing and improved save-to-file flow.

sudo
- Hardened lowercase conversion to avoid unsafe char handling.
- Updated unknown-option behavior to return a clear invalid-argument exit code.

system-events
- Added command options to filter logs:
  --count, --level, --provider, --since-hours, --no-message, --save, --help
- Added faster event querying and safer argument handling.
- Added provider metadata caching to reduce repeated expensive lookups.
- Fixed wide-string formatting and file-save string handling.

system-monitor
- Added --json output mode with uptime, memory load, process count, and process list.
- Refactored process collection into a helper function.
- Improved process access strategy by trying limited-query permissions first.

userinfo
- Hardened wide-string lowercase conversion using towlower.
- Added UTF-8 console output setup and locale initialization.

version
- Replaced fixed-size registry reads with dynamic buffer reads to avoid truncation.
- Fixed short-option parsing bounds check to prevent out-of-range access.

wget
- Added HTTP status code validation (requires successful 2xx response).
- Added network timeouts and retry attempts for transient failures.
- Added safer download flow using a temporary .part file, then atomic finalize/rename.
- Added stronger read/write failure handling and cleanup.
